Version control

Version control is about (well as the name states) controlling which version you are working on in a certain document/file. Version control is an improved and more secure way of keeping up to date on which file is the latest, which one was edited, which on was pulbished etc.

Git:
  • Git is a version control system.
  • Everything other version control systems are good at, git is better. In the sense that it has very very good performance
  • Git has both a local and a cloud repository, meaning you can store your files both locally and in the cloud. You can also work from either and then commit -> push it to the master branch.
Github:
  • Github is another version control system.
  • You can look at Github as "git made easy". In other words you can use github more easily, and its more of a social platform than only version control like git.
  • Github has both a local and a cloud repository, which again means that working from either is entirely possible, and you can also push/commit etc.
